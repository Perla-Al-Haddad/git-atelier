
## Cloner un dépot Git

Le clonage d'un dépot git reviens à télécharger localement un dépot qui existe déja sur **GitHub**. Ce téléchargement implique tous les fichiers, toutes les branches et tou sles commits du dépot.

### Comment cloner un dépot Git

#### Via une ligne de commande

Pour cloner un dépot git, utiliser la commande suivante dans le terminal:
``` bash
git clone [url]
```
L'url du dépot se situe sur la dépot github du projet. Il faut cliquer sur la flèche détoulante de l'onglet *code*, comme sur l'image suivante. 

![image](/git-atelier/images/url_depot_git.png)

#### Via un éditeur de code

Les éditeurs de code tels que **Visual Studio** et **Visual Studio Code** permette de cloner un dépot git. Ces derniers vous permettent de soit saisir l'url comme dans le terminal (emplacement du dépot), soit de se connecter à son compte github (icone github) et d'ensuite choisir un dépot à cloner.

## Les branches Git

### Concept des branches

Les branches sont utilisé afin de coder en parallèle avec des collaborateurs. Elles sont visible comme des "lignes de développement". Elles permettent de faire des commits en ne changeant que sa "ligne" sans impacter le travail des autres. 


### Les branches dans un projet

A la création d'un dépot git, une branche est créer, la branche **main** ou **master**. C'est cette branche qui contient le code qui est executé. Cela veut dire que ce qui est ajouter sur la branche **main** ne doit en aucun cas générer d'erreurs ou entrer en conflict avec du code existant car, soit le logiciel est en cours de conception et tous les autres collaborateurs ne pourront plus travailler à cause de cette erreur, soit le logiciel est déja déployé et cela causerait le crash du logiciel. Pour éviter cela, chaque collaborteur créer sa branche. Comme cela, s'il génère une erreur il est le seul impacté. Après avoir vérifié que son code est compatible avec la branche main, le dévveloppeur peut ajouter les modifications effectué sur sa branche à la branche **main**, c'est ce qu'on appelle un **merge**.  

Afin de nommer les branches, aucune règles n'est imposé par **Git**, mais des nomenclatures sont utilisé en entreprise afin de comprendre quel est l'intéret d'un branche. Cette nomenclature est *type-de-changement/fonction-en-elle-même*.  
Les types de changement souvent urilisé sont les suivant :  

* feature
* fix
* hotfix
* realease
* docs
* refactor
* test

Tant qu'a la parti de la fonction en elle même elle décrit ce que fit concrètement la fonction comme par exemple login, add-payment-method, etc... Il est préférable d'utiliser des tirets plutôt que des espaces ou des underscores et d'éviter les caractères spéciaux et les accents.

### Creér une branche

Pour créer une branche, rentrer la commande suivante dans le terminal :  
``` bash
git switch -c feature/fonction-nouvelle-feature
```

L'option *-c* permet de créer une nouvelle branche. Sans option, switch permet juste de changer de branche.

### Merge une branche

Pour merge une branche, rentrer la commande suivante :

``` bash
git switch main
git merge feature/fonction-nouvelle-feature
```

Il faut se rendre la branche où l'on souhaite merge la branche, dans ce cas la branche main, puis utiliser la fonction merge et indiquer la branche à merge.
