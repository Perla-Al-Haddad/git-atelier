Lorsque le développement sur une branche est terminé (ex: `feature/ISICG-132-save-json`), le travail n’est pas encore intégré au projet principal.

Il faut :

- Mettre sa branche à jour avec develop
- Vérifier qu’il n’y a pas de conflits
- Créer une Pull Request sur GitHub
- Demander une revue de code
- Corriger si nécessaire
- Merge la PR
- Supprimer la branche

## Quand créer une Pull Request ?
Une Pull Request sur GitHub peut être créée à n’importe quel moment :

- Quand la feature est terminée
- Quand elle est encore en cours
- Même dès le premier commit

En entreprise, il est courant de créer une PR :

- Pour signaler que le travail a commencé
- Pour obtenir un feedback précoce
- Pour rendre le travail visible à l’équipe

On appelle souvent cela une : ***Draft Pull Request***.

## Créer une Pull Request sur GitHub
Sur GitHub :

- Aller dans l’onglet Pull Requests
- Cliquer sur New Pull Request
- Sélectionner :
    - Base -> `develop`
    - Compare -> `feature/ISICG-132-save-json`
- Cliquer sur Create Pull Request

![PR](/images/PR.png)

### Contenu d’une bonne PR
Une PR doit contenir :

- Le lien vers le ticket (ex: ISICG-132)
- Une description claire
- Les changements effectués
- Des captures d’écran si nécessaire

## Revue de code

Dans l’interface GitHub, les reviewers peuvent :
- Commenter ligne par ligne
- Proposer des modifications
- Approuver la PR
- Demander des changements

On peut :

- Ajouter des reviewers
- Ajouter des labels

## Merge de la Pull Request
On commence par mettre notre branche a jour avec develop:
``` bash
# mettre a jour notre version de develop
git checkout develop
git fetch
git rebase develop

# mettre a jour notre branche
git checkout feature/ISICG-132-save-json
git fetch
git rebase feature/ISICG-132-save-json

# prendre tout les nouveaux commit de develop dans notre branche
git checkout feature/ISICG-132-save-json
git rebase -i develop

# pousser le code final sur la branche remote
git push origin feature/ISICG-132-save-json --force-with-lease
```

Puis sur GitHub sur la page de notre PR on click sur ***Merge Pull Request***. 

Sur GitHub, plusieurs options existent :
- Merge commit
- Squash and merge
- Rebase and merge

La plupart des équipes utilisent :

- Squash and merge (pour garder un historique propre)
