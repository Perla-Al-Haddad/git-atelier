Dans les projets open source, les développeurs n'ont généralement pas accès en écriture au dépôt principal. Au lieu de collaborer directement via des branches partagées, ils utilisent le modèle du fork. Ce modèle permet à chacun de travailler librement sur sa propre copie du projet, puis de proposer ses modifications au dépôt d'origine.

Comprendre le fonctionnement des forks est essentiel pour participer à des projets publics sur GitHub.

## Qu'est-ce qu'un fork ?
Un fork est une copie complète d'un dépôt GitHub, créée sous votre propre compte. Le fork est visible sur GitHub comme un nouveau dépôt, relié au projet original. Le dépôt original est souvent appelé `upstream`, tandis que votre copie personnelle devient votre dépôt principal de travail.

## Mise en pratique
### Étape 1 – Forker un dépôt
Ouvrir le dépôt principal sur GitHub
1. Cliquer sur le bouton Fork
2. Sélectionner votre compte
3. GitHub crée alors une copie du projet dans votre espace personnel.

### Étape 2 – Cloner votre fork
Une fois le fork créé, vous ne clonez pas le dépôt original, mais votre propre fork :
``` bash
git clone https://github.com/votre-username/nom-du-depot.git
cd nom-du-depot
```

À ce stade :
- origin pointe vers votre fork
- Le lien vers le dépôt original n'est pas encore configuré

### Étape 3 – Ajouter le dépôt original comme upstream
Pour pouvoir synchroniser votre fork avec le projet principal, vous devez ajouter un remote nommé `upstream` :
``` bash
git remote add upstream https://github.com/original-owner/nom-du-depot.git
``` 

Vérifiez :
``` bash
git remote -v
``` 

Vous devriez voir :
```
origin -> votre fork
upstream -> dépôt original
```

### Étape 4 – Synchroniser votre fork

Si le dépôt original évolue, votre fork doit être mis à jour.

Récupérer les changements :
``` bash
git fetch upstream
```
Mettre à jour votre branche principale :
``` bash
git checkout main
git rebase upstream/main
```

Mettre à jour votre fork sur GitHub :
``` bash
git push origin main
```
Votre fork est maintenant synchronisé avec le projet original.
