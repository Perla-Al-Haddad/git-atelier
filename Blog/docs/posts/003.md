## Modifications et création de commits

### Défintion d'un commit
Un commit est une sauvegarde du projet à un instanté. Ce dernier est acompagné d'un message permettant de correctement structurer un projet. Afin de justement avoir une bonne structure, il y a de bonnes pratiques à mettre en place :

* Faire un commit par modifications dans un projet. Cela permet de pouvoir récupérer précisément une version du projet.
* Faire des messages de commit explicite. Préciser le type de changment que l'on a fait (comme pour les branches) et expliquer brièvement à quoi cela correspond, exemple: feature : ajout page de connexion. Cela permet encore une fois de facilement s'y retrouver dans son historique de commit.

### Création de commit

Pour créer un commit, rentrer la commande suivante dans le terminal :
``` bash
git commit -m "[nom développeur] type de commit : explication du changement"
```

Seulement, afin de commit des fichiers, il faut les ajouter dans la *staging area*. Pour cela, il faut utiliser la commande suivante :

``` bash
git add <chemin relatif de fichier>
```
ou alors pour ajouter tous les fichiers qui ont été modifiés à la **staging area**, on utilise :

``` bash
git add .
ou
git add -A
```

Pour l'instant, tous les commits effectué ne sont enregistré que localement. Afin de les rendre visible sur l'espace de dépot github, il faut utiliser la commande :

``` bash
git push origin nom-branche
```

Avec *origin* le nom du dépot local (nom de base d'un dépot cloné) et nom-branche le nom de la branche sur laquelle les commits sont push. La commande git push sans argument fera un push du dépot local courant sur la branche courante si le dépot loacal est déja lié à la branche.

### Vérification avant commit

Pour vérifier quelles sont les fichiers affectés par un commit, on utilise la commande :

``` bash
git status
```

Cette commande permet de savoir : 
* Quelle est la branche actuelle
* Les fichiers modifiés dans la **staging area** et donc prêt à être commit.
* Les fichiers modifiés mais pas dans la **staging area**
* Les nouveaux fichiers créer mais pas dans la **staging area**

Pour ajouter un fichier à la **staging area**, vous savez déja faire avec git add, mais pour en retirer il faut utiliser la commande :
``` bash
git restore --staged <fichier>
```
