Le **contrôle de versions** permet de suivre l’évolution d’un projet informatique dans le temps. Il permet de :

* enregistrer les modifications du code,
* revenir à une version précédente,
* travailler à plusieurs sans perdre de changements.

Avant Git, des outils comme **CVS** et **Subversion** utilisaient un modèle **centralisé**, avec un serveur unique, peu adapté au travail collaboratif moderne.

**Git**, créé en 2005 par **Linus Torvalds**, est un système de contrôle de versions **distribué** :

* chaque développeur possède une copie complète du projet,
* le travail peut se faire hors ligne,
* les branches sont rapides et légères.

**Git** est l’outil de versionnement.
**GitHub** est une plateforme qui héberge des dépôts Git et facilite la collaboration (Pull Requests, revues de code, CI/CD).

Aujourd’hui, Git et GitHub sont des **outils standards** du développement logiciel, utilisés en entreprise et dans l’open source.
