Dans cette dernière section, vous allez découvrir comment transformer votre code versionné avec Git en un site web publié automatiquement. Cela relie directement le contrôle de versions à un processus de déploiement continu — une compétence essentielle en développement moderne.

GitHub Pages est un service de GitHub qui permet d'héberger gratuitement des sites statique (HTML, CSS, JavaScript) directement à partir d'un dépôt. C'est parfait pour publier un portfolio, un site d'application front-end ou une documentation.

## Préparer votre projet pour GitHub Pages

Modifier le fichier `vue.config.js` à la racine du projet pour que les fichiers statiques soient chargés correctement :
``` js
module.exports = {
  publicPath: process.env.NODE_ENV === "production" ? "/NOM_DU_REPO/" : "/",
};
```
Cette configuration indique à votre application où trouver ses ressources une fois publiée.

## Construire et déployer manuellement
Une fois votre projet prêt :

### 1. Construisez l'application pour la production

``` bash
npm run build
``` 
Cela génère un dossier (docs, build, etc.) contenant tous les fichiers statiques à publier.

### 2. Commitez ce dossier

Assurez-vous que le dossier de production n'est pas dans `.gitignore`, puis :
``` bash
git add dist
git commit -m "[PRENOM][Task-ID] Préparation du build pour GitHub Pages"
```

### 3. Publiez sur GitHub Pages via la branche gh-pages

GitHub Pages peut servir le contenu d'une branche dédiée, typiquement gh-pages. Une des manières traditionnelles consiste à utiliser la commande suivante :
``` bash
git subtree push --prefix docs origin gh-pages
```

Elle pousse le dossier de production (docs) comme une branche gh-pages, qui sera ensuite servie comme site.

Après quelques minutes, votre projet devrait être accessible à l'URL :
```
https://VOTRE_IDENTIFIANT.github.io/NOM_DU_REPO/
```
